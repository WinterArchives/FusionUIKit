I'm developing a View extensions with the TreeDataProvider. Users are able to right click on an item in the tree, opening the showOpenDialog window. There they can pick an svg or png image file to replace the current icon used for the selected Tree Item. Once chosen the new image overwrites the current icon file with the same name in the same path (same URI). Then the TreeDataProvider is refreshed using the onDidChangeTreeData event. On refresh the new icon does not load. Icons with different URI's will reload. The new icon only appears after a window reload. Am I missing a call in the API to force reload the URI, otherwise this would seem like a bug.
